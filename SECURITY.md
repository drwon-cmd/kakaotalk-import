# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

보안 취약점을 발견하셨다면 아래 절차를 따라주세요.

### 보고 방법

1. **공개 이슈로 보고하지 마세요** - 취약점이 악용될 수 있습니다.
2. **이메일로 보고**: drwon@wiltcm.com
3. **포함할 정보**:
   - 취약점 설명
   - 재현 단계
   - 영향 범위
   - 가능한 경우 수정 제안

### 응답 시간

- **확인**: 48시간 이내
- **초기 평가**: 7일 이내
- **수정 배포**: 심각도에 따라 결정

## 개인정보 보호 주의사항

이 프로젝트는 카카오톡 대화를 처리합니다. 사용 시 주의사항:

### 절대 커밋하지 마세요
- `raw/` 폴더의 원본 대화 파일 (.txt)
- `summaries/` 폴더의 요약 파일 (.md, .pdf)
- 개인정보가 포함된 모든 파일

### .gitignore 확인
`.gitignore` 파일이 다음을 제외하는지 확인하세요:
```
raw/**/*.txt
summaries/**/*.md
summaries/**/*.pdf
```

### Fork 시 주의
이 저장소를 Fork하기 전에 개인 대화 파일이 포함되어 있지 않은지 확인하세요.

## 보안 모범 사례

1. **로컬에서만 실행**: 대화 파일을 클라우드에 업로드하지 마세요
2. **정기 삭제**: 분석 완료 후 원본 파일 삭제 권장
3. **접근 제한**: raw/, summaries/ 폴더 접근 권한 제한
